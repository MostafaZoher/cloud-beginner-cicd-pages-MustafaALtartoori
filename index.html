What is CI/CD?

CI/CD stands for Continuous Integration and Continuous Delivery/Deployment. Continuous Integration means merging code changes frequently and running automated checks to catch problems early. Continuous Delivery/Deployment then automates packaging and releasing the validated code. It reduces manual steps, lowers deployment risk, and helps teams ship updates faster. In cloud systems, CI/CD is essential for reliable and repeatable releases.

My Pipeline

After I push to the main branch, GitHub Actions triggers a workflow automatically. It checks that the required files exist, uploads the website files, then deploys them to GitHub Pages. Once the workflow succeeds, the live site updates instantly with the new version.
